# La Santa - Landing Page Konzept (Enhanced Version)

Eine **massiv erweiterte**, elegante und dramatische Webseite im Dunkel/Gold Design als Premium-Konzept für die Kartell-Präsentation von **La Santa**.

## 🎨 Design

- **Farben**: Dunkel (Primär) / Gold (Akzent) / Rot (Verrat) / Grün (Wiedergeburt)
- **Stil**: Modern, edel, dramatisch, mit authentischem kolumbianischem Kartell-Flair
- **Sprache**: Deutsch mit spanischen Akzenten und Begriffen
- **Umfang**: **1000+ Zeilen HTML**, **2000+ Zeilen CSS**, **400+ Zeilen JavaScript**

### 📖 **Dramatische Geschichte (5 Kapitel)**
Die komplette Historie von La Santa:

## 📝 Anpassungen

### **Farben ändern**

In `styles.css` - `:root` Variablen:

```css
:root {
    --gold-primary: #DEINEGOLD;
    --dark-primary: #DEINDUNKEL;
}
```

### **Inhalte ändern**

Alle Texte in `index.html` - Such-Funktion nutzen für schnelle Anpassungen.

### **Neue Systeme hinzufügen**

Kopiere eine bestehende `.sistema-card` in `index.html` und passe an.

### **Ränge anpassen**

Rang-Definitionen befinden sich in den entsprechenden Sections der Jerarquía.

## 🌐 Deployment

### **Standalone (ohne Docker)**

```bash
# Nginx
cp index.html styles.css script.js /usr/share/nginx/html/
service nginx restart

# Apache
cp index.html styles.css script.js /var/www/html/
service apache2 restart
```

### **Docker (Empfohlen)**

```bash
# Image bauen
docker build -t la-santa-landing:v2 .

# Container starten
docker run -d -p 8080:80 --name la-santa la-santa-landing:v2
```

### **Docker Compose (Beste Option)**

```bash
# Production mit Auto-Restart
docker-compose -f docker-compose.prod.yml up -d

# Logs checken
docker-compose -f docker-compose.prod.yml logs -f

# Stoppen
docker-compose -f docker-compose.prod.yml down
```

## 🔧 Troubleshooting

### **Port bereits belegt**

```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <PID> /F

# Linux/Mac
lsof -ti:8080 | xargs kill -9
```

### **Docker Probleme**

```bash
# Container neu bauen
docker-compose build --no-cache
docker-compose up -d

# Logs anschauen
docker-compose logs -f
```

### **Animationen laufen nicht**

- Browser-Cache leeren (Ctrl+Shift+Del)
- Hard Reload (Ctrl+Shift+R)
- JavaScript Console auf Fehler prüfen
